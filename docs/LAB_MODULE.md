# Lab Module

## Scope

Phase 11 adds a clinic-scoped lab workflow for:

- lab test catalog management
- consultation-linked lab orders
- lab order status tracking
- structured lab reports
- abnormal flag detection
- patient lab history integration
- audit logging

## Backend Components

- `backend/src/modules/labs/labTest.model.js`: lab catalog schema and indexes
- `backend/src/modules/labs/labOrder.model.js`: consultation-linked lab order schema and status enums
- `backend/src/modules/labs/labReport.model.js`: structured report schema with abnormal flags and assistive summary storage
- `backend/src/modules/labs/lab.validator.js`: Zod request validation for catalog, order, report, and patient-history endpoints
- `backend/src/modules/labs/lab.repository.js`: scoped data-access helpers for tests, orders, and reports
- `backend/src/modules/labs/lab.service.js`: clinic scoping, business rules, abnormal detection, status transitions, and audit hooks
- `backend/src/modules/labs/lab.controller.js`: HTTP handlers
- `backend/src/modules/labs/lab.routes.js`: protected route registration
- `backend/src/common/utils/generateLabOrderNumber.js`: `LAB-YYYYMMDD-XXXX` generator using the shared sequence helper

- `POST /api/v1/labs/tests`

- `POST /api/v1/labs/orders`
- `GET /api/v1/labs/orders`
- `GET /api/v1/labs/orders/:id`
- `PATCH /api/v1/labs/orders/:id/status`
- `POST /api/v1/labs/reports`
- `GET /api/v1/labs/reports/:id`
- `PATCH /api/v1/labs/reports/:id`
- `PATCH /api/v1/labs/reports/:id/finalize`

Frontend:

- `/labs/tests`
- `/labs/orders`
- `/consultations/:consultationId/labs/new`
- `/labs/orders/:id`
- `/labs/reports/:id`
- `/patients/:patientId/labs`

## Key Rules

- all lab queries are clinic-scoped in the backend
- doctors can only access their own lab orders and reports
- lab order numbers are generated by the backend
- abnormal flags are generated by the backend from `numericValue` and `normalRange`
- finalized reports cannot be edited in the current workflow
- the frontend never calls the AI service directly for Phase 11 lab analysis

## Known Limits

- one report per order in the current MVP
- no OCR/CV extraction from uploaded reports
- no LIS integration
- no lab invoice generation in this phase
- no notification workflow in this phase
